# 派活儿 Paihuo · 开发启动包

> **一句话**：浏览器侧边栏插件，帮不太会用 AI 的打工人——把任务倒进来，自动拆成清单，标注 AI 能帮多少、该用哪个工具、第一句话说什么；聊进度自动划卡；一键写日报/周报/月报。
> **形态红线**：Chrome MV3 sidePanel · 无后端无账号 · 数据全本机 · **BYOK**（用户自己的 API key，产品负责把「注册→创建→复制→粘贴」引导到傻瓜级，绝不内置开发者的 key）。

## 开发目标（Definition of Victory）

一个可在 Chrome/Edge 开发者模式安装的 **v0.1.0 zip**：

1. 用户走完「接上 AI，只需 3 步」向导（选平台→粘 key→连接选推理模型）后，能**真实**完成：倒活（文字/文件/截图）→ 拆解成带工具与提示词的任务卡 → 复制提示词跳转工具 → 对话汇报进度 → 生成日报/周报（可套公司模板）。
2. mock 模式（`VITE_PAIHUO_MOCK=1`）下有 Playwright e2e 全链路回归，一条 `pnpm e2e` 全绿。
3. 权限最小化、除用户所选 LLM 平台外零外部依赖。
4. 范围之外的一律不做（语音、自动填充、飞书版、云同步、深色、i18n——见 spec §9）。

## 整体规划（一页纸）

| 里程碑 | 任务 | 出口标准 |
|---|---|---|
| M0 地基 | T01-T04 脚手架 / 设计令牌与组件 / 状态层 / llmClient+mock | 面板壳跑通，单测绿 |
| M1 接 AI | T05 三步向导（注册引导/拉模型/推理推荐/测连接） | mock 下全流程可走，错误态友好 |
| M2 核心循环 | T06-T09 倒活附件 / 拆解链路 / 工具目录 / 筛选关联 | 核心价值可演示 |
| M3 盯活 | T10-T11 对话小派(function calling) / 总览页 | 「纪要发完了」自动划卡 |
| M4 报活 | T12 三种汇报 + 模板 | 五动词闭环 |
| M5 收口 | T13-T16 右键收集 / 打磨 / e2e / 打包 | zip 干净实例装上即用 |
| ⛔ 停点 | T17 真 key 冒烟 · T18 商店上架 | **必须先问用户** |

预估：M0→M5（mock e2e 全绿）约 4-6 个全职循环日；T17 之后按真实模型质量再迭代提示词。

## 怎么执行（给 Claude / Sonnet 5）

**启动指令（用户粘贴这句即可）：**

> 读 CLAUDE.md、docs/00-product-spec.md、docs/01-dev-plan.md，按 Goal 运行协议从 T01 开始持续循环，直到触发停点或全部完成。

循环规则（详见 `CLAUDE.md`）：取最低编号 ready 任务 → 简述方案 → TDD → **真实验证达 DoD**（UI 任务必须起真实扩展，禁止只跑单测宣称完成）→ 在 `docs/01-dev-plan.md` 勾掉并追加 `✅ 日期 验证方式` → git commit → 下一个。LLM 功能一律先跑 mock，真 key 只在 T17（先问用户要）。连败 3 次 / 要花钱 / 要发布 / 想超出 spec → 停下问用户。
